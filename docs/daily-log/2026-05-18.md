## 学んだこと

- `filter` を keyframes 内で指定すると、既存の `drop-shadow()` を上書きしてしまうことがある。
- `animation-fill-mode: both` は最終状態を保持するため、後続の状態 class と干渉しやすい。
- `body.is-page-loaded .tile-wrap` のような詳細度の高いセレクタは、状態 class より勝つことがある。
- `script` タグより後にある要素は、実行タイミングによっては JavaScript から取得できず `null` になる。
- `border-radius: 50%` だけでは正円にならず、元要素の幅と高さも同じである必要がある。
- CSS Grid の `grid-column` / `grid-row` 変更は、そのままでは滑らかにアニメーションしにくい。
- `function` は複数箇所で使う処理をまとめるための仕組みであり、閉じる処理のような共通処理に向いている。
- JavaScript で付ける class は、どの動作を担う class なのか責務を明確にして管理しないと、原因調査が難しくなる。
- 1つの class に複数の役割を詰め込みすぎると、不具合の原因が追いにくくなる。
