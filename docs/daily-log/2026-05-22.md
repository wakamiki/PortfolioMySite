## 学んだこと

- 状態classは、付与する対象を責務に合わせて選ぶと設計が分かりやすい。

- `classList.contains()` は、要素が特定の状態classを持っているか確認するために使える。
  - `isTileLocked()` はロック状態の確認を担当し、`shouldBlockTileAction(event)` は今回のクリックを止めるべきか判断する役割に分けた。

- クリックを止める処理では、閉じるボタンだけでなく、展開中タイル全体を除外する方が自然な場合がある。

- タイルの状態差は、透明度だけでなく動きや光り方の違いでも伝えられる。
  - UI上の「押せる / 今は起動しない」の違いを、触感として表現できた。

- `z-index` があるだけでなく、どの要素が前面でクリックを受けるかまで確認する必要がある。
  - 見えているボタンでも、別要素の下に潜っているとクリックできない。

- `display: none` は要素をレイアウトから消すため、横並びUIでは位置ずれの原因になる。
  - 見えない状態でも配置スペースを残したい場合は `visibility: hidden` が使える。

- Gridは、縦方向の役割を明確に分けたい時にも使いやすい。
  - `auto minmax(0, 1fr) auto` で、固定ヘッダー・スクロール領域・固定操作部を整理できる。

- 既存の完成度が高い資料がある場合、すべてをWeb側で再実装するより、表示目的に合わせて画像化して活用した方が効率的なことがある。

- 高解像度画像を縮小表示する場合、作成時の文字・余白・線幅は最終表示サイズから逆算する必要がある。
  - 760pxで作成し380pxで表示するなら、文字や余白は約2倍値で設計する。

- `a` タグの意味を残しながら演出を挟みたい場合は、`preventDefault()` で即遷移を止めてから、JavaScript側でタイミングを制御できる。
