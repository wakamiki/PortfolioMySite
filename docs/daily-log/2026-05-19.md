## 判断したこと

- class の作成判断と CSS の書き方ルールを明確にした。
- 通常の見た目・配置・構造は、できるだけ単独 class で管理する。
- 複合セレクタは極力減らすが、親の状態によって中の部品を変えるものは複合セレクタを使ってよい。
- `h2` や `p` を直接指定するより、意味のある class がある場合は `.tile-motion-text` などを使う。
- `work-` と `works-` は完全統一ではなく、意味で使い分ける。
- 一覧ページと詳細ページでサイズや役割が変わるものは、早めに class を分ける。
- 初期表示用の `--intro-delay` と、再配置・復帰用の `--return-delay` は別責務として分ける。
- 外部リンクへ遷移する overlay は、遷移前に状態 class を外さず保持する。

## 学んだこと

- class 名は見た目だけでなく、どの責務を持つかを表す設計情報になる。
- 1つの class に複数の役割を詰め込むと、不具合原因の特定が難しくなる。
- CSS の複合セレクタには、残した方がよいものと、単独 class に分けた方がよいものがある。
- `.parent .child` は、親の状態によって内部パーツを制御する場合には有効。
- `A + B` は隣接兄弟セレクタで、直前に A がある B にだけ効く。
- `:active` はクリックやタップで押している最中の状態を表す。
- CSS変数も class と同じように責務を分ける必要がある。
- `--intro-delay` のような初期表示用の値を別アニメーションに使い回すと、意図しないタイミングのズレが起きる。
- 外部遷移アニメーションでは、ページ遷移まで overlay を保持する方が自然に見える。
