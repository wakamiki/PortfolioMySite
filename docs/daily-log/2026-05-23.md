## 学んだこと

- GitHub Pagesは設定直後にすぐ反映されない場合があり、数分からしばらく待つ必要がある。
  - `Your GitHub Pages site is currently being built from the main branch.` と出ている場合は、未設定ではなくビルド中と考える。

- Saveボタンが押せない場合でも、設定が失敗しているとは限らない。
  - すでに同じ設定が保存済みの場合、変更がないためSaveできない。

- `querySelector` は最初の1件だけを取得する。
  - 同じclassを持つ要素が複数ある場合、すべてに処理を付けたいなら `querySelectorAll` を使う。

- `data-*` 属性を使うと、HTML上の要素とJavaScriptの状態管理を対応させやすい。
  - `data-work="manual"` と `data-work-detail="manual"` のように対応関係を持たせることで、作品ごとの表示切り替えが分かりやすくなる。

- 同じスクロール領域を使い回す場合、前回の `scrollTop` が残ることがある。
  - 詳細ページを開くタイミングで `scrollTop = 0` にすると、常に先頭から表示できる。

- 公開用サムネイルでは、ローカルパスなど個人の作業環境が見える情報はマスキングした方がよい。
