## 学んだこと

- スマホやタブレットでは `hover` が効かず、`:active` も不安定になるため、必要に応じてJavaScriptで専用classを付けて制御する。
- `filter` は一部だけ追加されるのではなく、指定全体が上書きされる。
  - `brightness()` や `saturate()` を追加したい場合でも、残したい `drop-shadow()` や `blur()` も一緒に書く必要がある。
- `box-shadow` と `filter: drop-shadow()` は別物で、角落としタイルの形に沿う影には `drop-shadow()` が有効。
- タイルの枠SVGは、タイルの縦横比が変わるとずれやすい。
  - `grid-template-columns` と `grid-auto-rows` を同じ基準で管理すると、比率を保ちやすい。
- `clamp()` は便利だが、幅と高さの基準が別々になると比率維持には不十分な場合がある。
- モバイル実機確認では、DevToolsだけでは分からない長押し・タップ・戻る操作の不具合が見つかる。
- `user-select: none`、`-webkit-user-select: none`、`-webkit-touch-callout: none` は、操作UIとしてのタイルでは有効に使える。
