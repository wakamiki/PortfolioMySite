自己紹介

もっと良くできるのに、そのままになっているものが気になります。

使うたびに少し面倒な操作。
誰かに聞かなければ進められない仕組み。
便利ではあるけれど、触れていて心地よくないもの。

そうした小さな引っかかりを見つけると、
どうすればもっと自然に、もっと気持ちよく使えるのかを考えたくなります。

私が作りたいのは、使う人に努力や我慢を求めるのではなく、
その人が自分のやり方で迷わず目的へ進める仕組みです。

目が悪い人にとってのメガネのように、
特別な支援として構えるものではなく、
日常の中に自然に馴染み、
できることや選べることを少しずつ広げてくれるもの。
業務の中の不便を整えるシステムも、
触った感覚まで考えたWebサイトも、
物に触れることから体験が始まるプロダクトも、
私にとっては同じところから始まっています。

誰かが感じている不便や物足りなさを見つけ、
技術とデザインを使って、
自然に使い続けたくなる体験へ変えていくこと。

このサイトには、そのために考え、作り、試してきたものと、
これから形にしていくものを綴っていきます。

  • エラー文抽出・解説CLIツール


    【開発中】ErrorInsight


    エラーログから重要箇所を抽出し、原因調査を支援するPython CLIツール

    開発中プロジェクト Python CLIツール 長いエラーログからエラー分を抽出し、見やすく整形・エラー内容を解説してくれるCLIツールです。

  • プロジェクト概要

    長いエラーログから読むべき行を抽出し、原因候補と確認ポイントを日本語で表示するローカルCLIツールです。
    英語のエラー文や長いスタックトレースに慣れていない初学者でも、まず確認すべき箇所を素早く見つけられることを目的に開発しています。
    現在は Python / Java / C# / JavaScript / SQL
    の代表的なエラーを対象に、クリップボードにコピーしたログをもとに簡易分析を行います。外部送信なしでローカル環境のみで動作します。
